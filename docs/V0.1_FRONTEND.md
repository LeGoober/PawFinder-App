### State Management (AuthCubit)

The `AuthCubit` manages all authentication state:

| State | Description |
|-------|-------------|
| `AuthInitial()` | Not authenticated, no action in progress |
| `AuthLoading()` | Auth request in flight |
| `AuthAuthenticated(user)` | Successfully authenticated |
| `AuthCodeSent(phoneNumber)` | SMS code sent (legacy phone flow) |
| `AuthError(message)` | Auth failed with error message |
| `AuthVerificationFailed(message)` | SMS code verification failed |

**Key Methods:**
- `signInWithGoogle()` — Google OAuth → backend login → JWT storage
- `signUpWithEmail(email, password, displayName)` — Email register
- `signInWithEmail(email, password)` — Email login
- `checkAuthStatus()` — Restore session from stored token on app launch
- `logout()` — Clear session, sign out Google, delete token
- `signInDev()` — Dev-only bypass using `dev-user-0001`

**Token Storage:** JWT stored via `AuthService`, persisted in browser localStorage for session continuity across tab refreshes.

## 🏗️ Build Pipeline

### Docker Multi-Stage Build

```dockerfile
# Stage 1: Build Flutter Web
FROM ghcr.io/cirruslabs/flutter:3.41.6 AS build
COPY pubspec.yaml pubspec.lock* ./
RUN flutter pub get
COPY . .
RUN flutter build web --release --no-tree-shake-icons

# Stage 2: Serve with Nginx
FROM nginx:1.27-alpine
COPY --from=build /app/build/web /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

### Nginx Configuration

```
Server: nginx/1.27-alpine
Port: 80
Health: GET /
```

**Features:**
- **SPA fallback:** `try_files $uri $uri/ /index.html` — all non-file routes serve `index.html` for client-side routing
- **Gzip compression:** CSS, JS, JSON, XML, SVG (min 1024 bytes)
- **Security headers:** `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`
- **Static asset caching:** `/assets/` → 1 year, immutable
- **Hidden file protection:** Blocks access to `/.` paths

## 🌐 CORS & API Communication

The frontend connects to the **API Gateway**, not directly to backend services:

```
Frontend (Nginx) ──HTTPS──► API Gateway (:8080) ──► Backend Services
```

**API Base URL:** `https://pawfinder-api-gateway.onrender.com`

All API calls go through a custom `ApiClient` with:
- JWT interceptor (automatically attaches `Authorization: Bearer <token>`)
- Token refresh interceptor (catches 401, attempts refresh, retries)
- Base URL configured from `api_constants.dart`

## 🔧 Known Limitations (V0.1)

1. **Web-only:** No iOS/Android builds deployed — Flutter Web only for V0.1
2. **No PWA:** Service worker / offline support not yet configured
3. **Google Sign-In popup:** May be blocked by some browsers; user must allow popups
4. **Cold start:** Free tier Nginx sleeps after 15 min inactivity — first load takes 30-60s
5. **Static env:** Google Client ID is baked into `web/index.html` — must rebuild to change
6. **No SSR/SEO:** Flutter Web renders entirely client-side; no server-side rendering
