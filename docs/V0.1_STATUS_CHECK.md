## 🧪 Full Test Procedure

### Step 1: Gateway Health Check

**Purpose:** Verify the API Gateway is running and can respond.

```bash
curl -s https://pawfinder-api-gateway.onrender.com/actuator/health | jq .
```

**Expected Response:**
```json
{
  "status": "UP"
}
```

**If this fails:**
- Wait 30-60 seconds (cold start wake-up)
- Check Render Dashboard: https://dashboard.render.com → pawfinder-api-gateway → Logs
- Verify the service hasn't crashed or exceeded free tier limits

---

### Step 2: Auth Service Health Check

**Purpose:** Verify the Auth Service is running independently of the gateway.

```bash
curl -s https://pawfinder-auth-service.onrender.com/actuator/health | jq .
```

**Expected Response:**
```json
{
  "status": "UP",
  "components": {
    "db": { "status": "UP" },
    "flyway": { "status": "UP" }
  }
}
```

**If `db` is DOWN:**
- Check `SPRING_DATASOURCE_URL`, `SPRING_DATASOURCE_USERNAME`, `SPRING_DATASOURCE_PASSWORD` env vars
- Verify pawfinder-db is running on Render
- Check database connection string resolves from Render's internal network

**If `flyway` is DOWN:**
- Check Flyway logs for migration errors
- Verify migration SQL files are present in the built JAR

---

### Step 3: Auth Login — Direct (Auth Service)

**Purpose:** Verify the Auth Service can process login requests and issues valid JWTs.

```bash
curl -s -X POST \
  https://pawfinder-auth-service.onrender.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"authProvider":"GOOGLE","authId":"test-verify-v01-001"}' | jq .
```

**Expected Response (200 OK):**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIs...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
  "user": {
    "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "authProvider": "GOOGLE",
    "displayName": null,
    "verified": false,
    "rescuerBadgeLevel": 0
  }
}
```

**Validation checks:**
1. HTTP status is `200`
2. `accessToken` is present and non-empty
3. `refreshToken` is present and non-empty
4. `user.id` is a valid UUID
5. Response time is under 5 seconds (may be longer on first request due to cold start)

**If this fails:**
- Verify `test-verify-v01-001` is a valid test auth ID (or use a real Google auth ID)
- Check Auth Service logs in Render Dashboard for errors
- Verify `JWT_SECRET` is configured

---

### Step 4: Auth Login — Via API Gateway

**Purpose:** Verify the API Gateway correctly routes auth requests to the Auth Service.

```bash
curl -s -X POST \
  https://pawfinder-api-gateway.onrender.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"authProvider":"GOOGLE","authId":"test-verify-v01-002"}' | jq .
```

**Expected Response (200 OK):**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIs...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
  "user": {
    "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "authProvider": "GOOGLE",
    "displayName": null,
    "verified": false,
    "rescuerBadgeLevel": 0
  }
}
```

**Validation checks:**
1. Response is identical in structure to the direct Auth Service response (Step 3)
2. Gateway adds no additional wrapping or transformation
3. CORS headers are present: `Access-Control-Allow-Origin: *` (or configured origin)
4. Response time is ≤ Step 3 + gateway overhead (~50-100ms extra)

**If this fails but Step 3 succeeds:**
- Gateway route for `/api/v1/auth/**` may not be reaching the Auth Service
- Check `AUTH_SERVICE_URL` env var on the gateway
- Verify Auth Service URL is reachable from Render's internal network
- Check gateway logs: `org.springframework.cloud.gateway` at DEBUG level

---

### Step 5: Frontend Smoke Test

**Purpose:** Verify the Flutter Web frontend is served correctly.

```bash
curl -sI https://pawfinder-frontend-juem.onrender.com
```

**Expected Response:**
```
HTTP/1.1 200 OK
Server: nginx/1.27.x
Content-Type: text/html
X-Frame-Options: SAMEORIGIN
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
```

**Validation checks:**
1. HTTP status is `200`
2. Server is `nginx`
3. Security headers present: `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`
4. Content-Type is `text/html`

**Manual browser test:**
1. Open: https://pawfinder-frontend-juem.onrender.com
2. Should see the PawFinder splash screen with animated paw logo
3. After ~2 seconds, should redirect to onboarding/login
4. Click "Sign in with Google" — OAuth popup should appear
5. Complete Google Sign-In — should redirect to Home page

**If this fails:**
- Wait 30-60 seconds (Flutter Web build + Nginx cold start)
- Check if the Flutter build succeeded (Render build logs)
- Verify `web/index.html` has the correct Google Client ID in the meta tag

### Step 6: Authenticated Request Flow

**Purpose:** End-to-end test: login → get token → call protected endpoint.

```bash
# 1. Login and capture the access token
TOKEN=$(curl -s -X POST \
  https://pawfinder-api-gateway.onrender.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"authProvider":"GOOGLE","authId":"test-e2e-v01-001"}' \
  | jq -r '.accessToken')

echo "Token: ${TOKEN:0:20}..."

# 2. Call /me endpoint with the token
curl -s \
  https://pawfinder-api-gateway.onrender.com/api/v1/auth/me \
  -H "Authorization: Bearer $TOKEN" | jq .
```

**Expected Response (from `/me`):**
```json
{
  "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "authProvider": "GOOGLE",
  "authId": "test-e2e-v01-001",
  "displayName": null,
  "verified": false,
  "rescuerBadgeLevel": 0,
  "createdAt": "2026-06-25T...",
  "lastActive": "2026-06-25T..."
}
```

**If `/me` fails:**
- Check JWT is valid and not expired
- Verify the token was properly captured from the login response
- Try with `curl -v` to see headers and debug auth errors

---

### Step 7: Token Refresh Flow

**Purpose:** Verify refresh tokens work.

```bash
# 1. Login and capture both tokens
RESPONSE=$(curl -s -X POST \
  https://pawfinder-api-gateway.onrender.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"authProvider":"GOOGLE","authId":"test-refresh-v01-001"}')

ACCESS_TOKEN=$(echo $RESPONSE | jq -r '.accessToken')
REFRESH_TOKEN=$(echo $RESPONSE | jq -r '.refreshToken')

# 2. Refresh the token
curl -s -X POST \
  https://pawfinder-api-gateway.onrender.com/api/v1/auth/refresh \
  -H "Content-Type: application/json" \
  -d "{\"refreshToken\":\"$REFRESH_TOKEN\"}" | jq .
```

**Expected Response:**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIs...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIs..."
}
```

**Validation:**
- New `accessToken` is different from the original
- `refreshToken` is rotated (new value)

## 🧪 Automated Test Script

Save as `test_v01.ps1`:

```powershell
# PawFinder V0.1 — Health Check Script
# Usage: .\test_v01.ps1

$ErrorActionPreference = "Continue"
Write-Host "🐾 PawFinder V0.1 Health Check" -ForegroundColor Cyan
Write-Host "==============================" -ForegroundColor Cyan
Write-Host ""

# Colors
function Pass { Write-Host "  ✅ PASS" -ForegroundColor Green }
function Fail { Write-Host "  ❌ FAIL" -ForegroundColor Red }

# 1. Gateway Health
Write-Host "1. Gateway Health..." -NoNewline
try {
    $result = Invoke-RestMethod -Uri "https://pawfinder-api-gateway.onrender.com/actuator/health" -TimeoutSec 60
    if ($result.status -eq "UP") { Pass } else { Fail; Write-Host "    Status: $($result.status)" }
} catch { Fail; Write-Host "    Error: $($_.Exception.Message)" }

# 2. Auth Health
Write-Host "2. Auth Service Health..." -NoNewline
try {
    $result = Invoke-RestMethod -Uri "https://pawfinder-auth-service.onrender.com/actuator/health" -TimeoutSec 60
    if ($result.status -eq "UP") { Pass } else { Fail; Write-Host "    Status: $($result.status)" }
} catch { Fail; Write-Host "    Error: $($_.Exception.Message)" }

# 3. Auth Login (Direct)
Write-Host "3. Auth Login (Direct)..." -NoNewline
try {
    $body = @{ authProvider = "GOOGLE"; authId = "test-healthcheck-$((Get-Date).Ticks)" } | ConvertTo-Json
    $result = Invoke-RestMethod -Uri "https://pawfinder-auth-service.onrender.com/api/v1/auth/login" -Method Post -Body $body -ContentType "application/json" -TimeoutSec 60
    if ($result.accessToken) { Pass } else { Fail; Write-Host "    No accessToken in response" }
} catch { Fail; Write-Host "    Error: $($_.Exception.Message)" }

# 4. Auth Login (Gateway)
Write-Host "4. Auth Login (via Gateway)..." -NoNewline
try {
    $body = @{ authProvider = "GOOGLE"; authId = "test-gw-$((Get-Date).Ticks)" } | ConvertTo-Json
    $result = Invoke-RestMethod -Uri "https://pawfinder-api-gateway.onrender.com/api/v1/auth/login" -Method Post -Body $body -ContentType "application/json" -TimeoutSec 60
    if ($result.accessToken) { Pass } else { Fail; Write-Host "    No accessToken in response" }
} catch { Fail; Write-Host "    Error: $($_.Exception.Message)" }

# 5. Frontend
Write-Host "5. Frontend..." -NoNewline
try {
    $response = Invoke-WebRequest -Uri "https://pawfinder-frontend-juem.onrender.com" -TimeoutSec 60
    if ($response.StatusCode -eq 200) { Pass }
    else { Fail; Write-Host "    HTTP $($response.StatusCode)" }
} catch { Fail; Write-Host "    Error: $($_.Exception.Message)" }

Write-Host ""
Write-Host "Health check complete." -ForegroundColor Cyan
```

## 🔧 Troubleshooting Guide

### Issue: Service Returns 502/503

| Cause | Solution |
|-------|----------|
| Cold start (service sleeping) | Wait 30-60 seconds, retry |
| Service crashed | Check Render Dashboard → Logs → Deploy |
| Out of memory | Check Render metrics; consider upgrading plan |
| Build failed | Check build logs in Render Dashboard |

### Issue: Auth Login Returns 401

| Cause | Solution |
|-------|----------|
| Invalid auth provider | Use `GOOGLE`, `EMAIL`, or `PHONE` |
| JWT_SECRET mismatch | Ensure same `JWT_SECRET` across all services |
| Token expired | Refresh token or re-login |
| User not found | First login creates user; check DB connection |

### Issue: Gateway Routes Not Working (but Direct Access Works)

| Cause | Solution |
|-------|----------|
| Wrong `SERVICE_URL` env var | Check `${SERVICE_NAME}_URL` is set correctly |
| Service not reachable | Verify service is deployed and healthy |
| DNS resolution failure | Services should use Render internal URLs |
| Circuit breaker open | Wait 10 seconds for breaker to half-open |
| Rate limiter tripped | Reduce request frequency (10 req/s replenish) |

### Issue: Redis Health Check Failing

| Cause | Solution |
|-------|----------|
| Render DNS can't resolve Upstash hostname | Already mitigated: `MANAGEMENT_HEALTH_REDIS_ENABLED=false` |
| Redis connection refused | Verify `REDIS_URL` env var is correct |
| Upstash instance down | Check Upstash console: https://console.upstash.com |

### Issue: Frontend Blank Page / Loading Forever

| Cause | Solution |
|-------|----------|
| Flutter JS failed to load | Check browser console for errors |
| Google Sign-In blocked | Allow popups for the domain |
| API unreachable | Check CORS headers; verify gateway URL in `api_constants.dart` |
| Stale build cache | Trigger manual redeploy in Render Dashboard |
| Cold start | Wait 30-60 seconds, refresh |

## ⏱️ Cold Start Timing Reference

First request after 15+ minutes of inactivity:

| Service | Typical Wake-up Time |
|---------|---------------------|
| pawfinder-api-gateway | 30-50 seconds |
| pawfinder-auth-service | 30-50 seconds |
| pawfinder-frontend | 20-40 seconds |
| pawfinder-db | Instant (managed) |
| Upstash Redis | Instant (always-on) |

> **Tip:** Run the health checks sequentially. If the first service is waking up, subsequent services may also be cold. Give each 60 seconds before concluding a failure.
