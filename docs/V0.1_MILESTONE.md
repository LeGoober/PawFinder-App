## 🎯 V0.1 Overview

PawFinder V0.1 is a **multi-service pet reunification platform**. The core infrastructure — authentication, API gateway, and a Flutter Web frontend — is live on Render. The architecture follows a microservices pattern with a centralized API Gateway routing requests to backend services, all backed by PostgreSQL + PostGIS and Upstash Redis.

### V0.1 Scope

> **V0.1 = Auth Service + API Gateway + Frontend live on Render. Backend services defined, Dockerfiles ready, deployable in hours.**

## 🔑 Key Decisions

| Decision | Rationale | Date |
|----------|-----------|------|
| **Render over AWS ECS** | Faster setup, free tier for prototyping, managed DB/Redis | 2026-06 |
| **Spring Cloud Gateway (WebFlux)** over Zuul/Kong | Native Spring Boot integration, reactive non-blocking | 2026-06 |
| **Upstash Redis** over Render managed Redis | Serverless pricing, global edge, no cold starts for cache | 2026-06 |
| **PostgreSQL + PostGIS** over MySQL/MongoDB | Geospatial queries essential for pet location | 2026-06 |
| **Per-service Dockerfiles** over build args | Render Blueprint doesn't support `--build-arg` | 2026-06 |
| **JWT auth** over session-based | Stateless, microservices-friendly, mobile-ready | 2026-06 |
| **Google Sign-In** as primary OAuth | Zero-friction onboarding, most users have Google | 2026-06 |
| **Flyway** over Hibernate DDL | Versioned, repeatable migrations; safe for production | 2026-06 |
| **Flutter Web** over React/Next.js | Single codebase for web + future mobile; strong Material Design | 2026-06 |
| **Mono-repo** over multi-repo | 9 services manageable in one repo for early stage | 2026-06 |
| **Render CLI/API deploy** over Blueprint auto-deploy | Per-service control, error visibility, partial deploys | 2026-06 |
| **Free tier (starter plan)** for V0.1 | Cost optimization while proving the concept | 2026-06 |
| **Disabled Redis health check** on management actuator | Mitigates Render DNS resolution issue with Upstash hostname | 2026-06 |
