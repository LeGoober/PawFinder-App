## Quick Launch (Right Now)

### What's already done
- вң… `main` branch has all 12 commits (functional prototype, auth, UI, deploy pipeline)
- вң… `render.yaml` Blueprint pushed вҖ” 10 services + DB + Redis defined
- вң… Dockerfiles for frontend (Flutter вҶ’ Nginx) and all backend services
- вң… `.github/workflows/deploy.yml` вҖ” CI/CD runs on push to main
- вң… Secrets stripped from git history (`docs/RENDER_ENV_VARS.txt` uses placeholders)

### What you need to do now

#### Step 1: Connect Render to GitHub
1. Go to https://dashboard.render.com
2. **Blueprints** вҶ’ **New Blueprint Instance**
3. Connect repository: `LeGoober/PawFinder-App`
4. Render auto-detects `render.yaml` and creates all 12 resources

#### Step 2: Set Secret Environment Variables
For every service marked `sync: false` in render.yaml, you must manually enter the real value in Render Dashboard:

| Service | Variable | Where to Get It |
|---------|----------|-----------------|
| **pawfinder-auth-service** | `TWILIO_ACCOUNT_SID` | Twilio Console |
| | `TWILIO_AUTH_TOKEN` | Twilio Console |
| | `TWILIO_PHONE_NUMBER` | Twilio Console |
| **pawfinder-reward-service** | `STRIPE_SECRET_KEY` | Stripe Dashboard (test mode: `sk_test_...`) |
| | `STRIPE_WEBHOOK_SECRET` | Stripe Dashboard вҶ’ Webhooks |
| **pawfinder-media-service** | `AWS_ACCESS_KEY_ID` | AWS IAM |
| | `AWS_SECRET_ACCESS_KEY` | AWS IAM |
| | `AWS_S3_BUCKET` | AWS S3 (e.g. `pawfinder-media-dev`) |
| **pawfinder-matching-service** | `AWS_ACCESS_KEY_ID` | (same as media) |
| | `AWS_SECRET_ACCESS_KEY` | (same as media) |
| **pawfinder-notification-service** | `FIREBASE_SERVER_KEY` | Firebase Console вҶ’ Cloud Messaging |
| | `SENDGRID_API_KEY` | SendGrid Dashboard |

> рҹ’Ў `JWT_SECRET` is auto-generated by Render (`generateValue: true`). DB credentials, Redis URL, and inter-service URLs are auto-wired via `fromService`/`fromDatabase`.

#### Step 3: Enable PostGIS on the Database
1. After `pawfinder-db` is provisioned, open its Shell tab in Render
2. Run:
   ```sql
   CREATE EXTENSION IF NOT EXISTS postgis;
   ```

#### Step 4: Configure Google OAuth
1. Go to https://console.cloud.google.com вҖ” APIs & Services вҶ’ Credentials
2. Edit your OAuth 2.0 Client ID
3. Add to **Authorized JavaScript origins:**
   ```
   https://pawfinder-frontend.onrender.com
   ```
4. Add to **Authorized redirect URIs:**
   ```
   https://pawfinder-frontend.onrender.com
   ```
5. If not already done, enable the **People API** (see `docs/GOOGLE_AUTH_SETUP.md`)

#### Step 5: Verify Everything
See [Post-Deploy Verification](#post-deploy-verification) below.

## Local Development

### Prerequisites
- Docker Desktop
- Java 21
- Flutter SDK 3.24+
- PowerShell (Windows)

### 1. Start Infrastructure

```powershell
cd pawfinder-app-backend
docker compose -f docker-compose.yml up -d
```

This starts PostgreSQL+PostGIS, Redis, RabbitMQ, Elasticsearch.

### 2. Start Backend Services

```powershell
# All services together
.\start-all.ps1
```

Services and ports:

| Service | Port | Key Dependency |
|---------|------|----------------|
| API Gateway | 8080 | Redis (rate limiting) |
| Auth Service | 8081 | PostgreSQL, Twilio |
| Alert Service | 8082 | PostgreSQL+PostGIS |
| Messaging | 8083 | PostgreSQL |
| Reward | 8084 | PostgreSQL, Stripe |
| Media | 8085 | AWS S3 |
| Analytics | 8086 | PostgreSQL |
| Matching | 8087 | AWS Rekognition |
| Notification | 8088 | Firebase, SendGrid |

### 3. Start Flutter Web Frontend

```powershell
cd pawfinder_app_frontend
flutter pub get
flutter run -d chrome --web-port 9876
```

### 4. Docker Build (production-like)

```powershell
# Frontend вҖ” Flutter build вҶ’ Nginx serve
cd pawfinder_app_frontend
docker build -t pawfinder-frontend .
docker run -p 8089:80 pawfinder-frontend

# Backend вҖ” per-service
cd pawfinder-app-backend
docker build --build-arg SERVICE_NAME=pawfinder-auth-service -t pawfinder-auth .
docker run -p 8081:8080 \
  -e SPRING_DATASOURCE_URL=jdbc:postgresql://host.docker.internal:5433/pawfinder \
  -e SPRING_DATASOURCE_USERNAME=pawfinder \
  -e SPRING_DATASOURCE_PASSWORD=pawfinder_dev \
  pawfinder-auth
```

## Render Deployment вҖ” Step by Step

### Prerequisites
- [x] Repo pushed to GitHub: `LeGoober/PawFinder-App`
- [x] `render.yaml` pushed to `main` branch
- [x] Dockerfiles in `pawfinder_app_frontend/` and `pawfinder-app-backend/`
- [ ] Render account connected to GitHub
- [ ] API keys ready (Twilio, Stripe, AWS, SendGrid, Firebase)

### Phase 1: Create the Blueprint

1. Go to https://dashboard.render.com
2. Click **Blueprints** in the left sidebar
3. Click **New Blueprint Instance**
4. Select `LeGoober/PawFinder-App`
5. Render reads `render.yaml` and shows all 12 resources it will create
6. Click **Apply**

> Render will begin building all services in parallel. Expect 5вҖ“15 minutes for initial builds.

### Phase 2: Configure Secrets

After services are created, go to each one and add the secret env vars:

1. Dashboard вҶ’ Select service вҶ’ **Environment** tab
2. Add each `sync: false` variable with its real value
3. Click **Save Changes** вҖ” triggers automatic redeploy

> **Reference:** Copy-paste from `docs/RENDER_ENV_VARS.txt` and replace the `PLACEHOLDER` values.

### Phase 3: Enable PostGIS

1. Dashboard вҶ’ `pawfinder-db` вҶ’ **Shell** tab (or connect via external tool)
2. Run:
   ```sql
   CREATE EXTENSION IF NOT EXISTS postgis;
   ```
3. Verify: `SELECT PostGIS_Version();` should return a version like `3.4`

### Phase 4: Google OAuth Setup

1. Google Cloud Console вҶ’ APIs & Services вҶ’ Credentials
2. Edit your OAuth 2.0 Client ID
3. Add **Authorized JavaScript origins:**
   ```
   https://pawfinder-frontend.onrender.com
   ```
4. Add **Authorized redirect URIs:**
   ```
   https://pawfinder-frontend.onrender.com
   ```
5. Verify **People API** is enabled (see `docs/GOOGLE_AUTH_SETUP.md`)

### Phase 5: Stripe Webhook (if using rewards)

1. Stripe Dashboard вҶ’ Developers вҶ’ Webhooks вҶ’ Add endpoint
2. URL: `https://pawfinder-reward-service.onrender.com/api/v1/rewards/webhook`
3. Events: `payment_intent.succeeded`, `checkout.session.completed`
4. Copy the signing secret вҶ’ set as `STRIPE_WEBHOOK_SECRET` in Render

## Environment Variables Reference

### Auto-Wired by Render (no manual setup needed)

| Variable | Source |
|----------|--------|
| `SPRING_DATASOURCE_URL` | `pawfinder-db` connection string |
| `SPRING_DATASOURCE_USERNAME` | `pawfinder-db` user |
| `SPRING_DATASOURCE_PASSWORD` | `pawfinder-db` password |
| `REDIS_URL` | `pawfinder-redis` connection string |
| `JWT_SECRET` | Auto-generated by Render |
| `AUTH_SERVICE_URL` | `pawfinder-auth-service` host:port |
| `ALERT_SERVICE_URL` | `pawfinder-alert-service` host:port |
| All other `*_SERVICE_URL` | Resolved via `fromService` |

### Manual Setup Required (sync: false)

| Service | Variable | Where From |
|---------|----------|------------|
| auth | `TWILIO_ACCOUNT_SID` | Twilio Console вҶ’ Account SID |
| auth | `TWILIO_AUTH_TOKEN` | Twilio Console вҶ’ Auth Token |
| auth | `TWILIO_PHONE_NUMBER` | Twilio Console вҶ’ Active Number |
| reward | `STRIPE_SECRET_KEY` | Stripe Dashboard вҶ’ API Keys |
| reward | `STRIPE_WEBHOOK_SECRET` | Stripe Dashboard вҶ’ Webhooks |
| media | `AWS_ACCESS_KEY_ID` | AWS IAM вҶ’ User security credentials |
| media | `AWS_SECRET_ACCESS_KEY` | AWS IAM вҶ’ User security credentials |
| media | `AWS_S3_BUCKET` | AWS S3 вҶ’ Bucket name |
| matching | `AWS_ACCESS_KEY_ID` | (same IAM user as media) |
| matching | `AWS_SECRET_ACCESS_KEY` | (same IAM user as media) |
| notification | `FIREBASE_SERVER_KEY` | Firebase Console вҶ’ Cloud Messaging |
| notification | `SENDGRID_API_KEY` | SendGrid вҶ’ API Keys |

### JWT_SECRET Consistency

`JWT_SECRET` must be the **same value** across: API Gateway, Auth, Alert, Messaging, Reward, Media, Analytics, Matching, Notification.

**Render handles this automatically** вҖ” it generates one value and syncs it across services (via `generateValue: true` in the first service + `sync: false` in subsequent ones). Verify in each service's Environment tab that `JWT_SECRET` has the same value.

## Post-Deploy Verification

### 1. Check all services are healthy

Each service has a health check at `/actuator/health` (or `/` for frontend):

```powershell
# Frontend
curl -sI https://pawfinder-frontend.onrender.com | head -1
# Expected: HTTP/1.1 200 OK

# API Gateway
curl https://pawfinder-api-gateway.onrender.com/actuator/health
# Expected: {"status":"UP"}

# Auth Service
curl https://pawfinder-auth-service.onrender.com/actuator/health
# Expected: {"status":"UP"}
```

### 2. Test the auth flow

1. Open `https://pawfinder-frontend.onrender.com`
2. You should see: Splash вҶ’ Onboarding вҶ’ Login page
3. Enter a phone number
4. Dev mode (local) sends code `123456` вҖ” **not available in prod**
5. In production, Twilio sends a real SMS with a 6-digit code
6. Enter the code вҶ’ redirected to Dashboard

### 3. Verify database connectivity

```powershell
# From Render Shell or external client
psql $DATABASE_URL
SELECT * FROM flyway_schema_history;
\d users
```

### 4. Check Flyway migrations ran

The auth service runs Flyway on startup. Check its logs:
- Render Dashboard вҶ’ `pawfinder-auth-service` вҶ’ Logs
- Look for: `Successfully applied migration` or `Schema is up to date`

### Common Issues

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| Frontend shows blank page | Nginx not serving SPA fallback | Check `nginx.conf` вҖ” should rewrite `/*` to `/index.html` |
| Auth returns 500 | PostGIS not enabled | Run `CREATE EXTENSION postgis;` on pawfinder-db |
| Auth returns 403 | People API not enabled | Google Cloud Console вҶ’ enable People API |
| `Connection refused` to backend | Inter-service URL wrong | Render auto-wires these вҖ” check service names match render.yaml |
| Build fails: `flutter: not found` | Frontend runtime is `static`, not `docker` | Fixed in render.yaml вҖ” use Docker runtime |
| Push rejected: secret scanning | API keys in committed files | Fixed вҖ” use `docs/RENDER_ENV_VARS.txt` with placeholders |

## Mobile Deployment Roadmap

### Android (Google Play Store)

| Step | Action | Complexity |
|------|--------|------------|
| 1 | Generate keystore (`keytool -genkey`) | Low |
| 2 | Create `key.properties` | Low |
| 3 | Configure `build.gradle` for signing | Low |
| 4 | Build release: `flutter build appbundle` | Low |
| 5 | Create Play Console account ($25 one-time) | Low |
| 6 | Upload `.aab` to Play Console | Low |
| 7 | Set up internal testing track | Low |

**From Windows:** Fully supported.

### iOS (Apple App Store)

| Step | Action | Complexity |
|------|--------|------------|
| 1 | Apple Developer account ($99/year) | Low (but cost) |
| 2 | **Acquire macOS machine** (or cloud Mac) | **HIGH** вҖ” cannot build iOS on Windows |
| 3 | Configure Xcode project + code signing | Medium-High |
| 4 | Build release: `flutter build ipa` | Medium |
| 5 | App Store review (1-3 days) | Medium |

**Options for iOS builds from Windows:**
1. **Codemagic** (recommended) вҖ” free tier, 500 min/month, handles signing + App Store upload
2. **GitHub Actions macOS runner** вҖ” $0.08/min, good for occasional builds
3. **Mac Mini** (~$600) вҖ” permanent solution if iOS is a core target
