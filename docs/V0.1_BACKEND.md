## 🌐 API Gateway Routing Table

All external traffic hits the **API Gateway** on port 8080. The gateway authenticates, rate-limits, and routes to backend services.

### Base URL

```
https://pawfinder-api-gateway.onrender.com
```

### Route Table

| Route ID | Path Pattern | Target Service | Target URI (prod) |
|----------|-------------|----------------|-------------------|
| `auth-service` | `/api/v1/auth/**` | Auth Service | `https://pawfinder-auth-service.onrender.com` |
| `alert-service` | `/api/v1/alerts/**`, `/api/v1/sightings/**`, `/api/v1/pets/**` | Alert Service | `${ALERT_SERVICE_URL}` |
| `messaging-service` | `/api/v1/conversations/**`, `/api/v1/messages/**` | Messaging Service | `${MESSAGING_SERVICE_URL}` |
| `reward-service` | `/api/v1/rewards/**` | Reward Service | `${REWARD_SERVICE_URL}` |
| `media-service` | `/api/v1/media/**` | Media Service | `${MEDIA_SERVICE_URL}` |
| `analytics-service` | `/api/v1/dashboard/**`, `/api/v1/leaderboard/**`, `/api/v1/badges/**` | Analytics Service | `${ANALYTICS_SERVICE_URL}` |
| `matching-service` | `/api/v1/matching/**` | Matching Service | `${MATCHING_SERVICE_URL}` |
| `notification-service` | `/api/v1/notifications/**` | Notification Service | `${NOTIFICATION_SERVICE_URL}` |

### Gateway Features

- **Rate Limiting:** 10 req/s replenish, 20 burst (auth route only; configurable per route)
- **Circuit Breakers:** Resilience4j — 50% failure threshold, 10s open state, 10-call sliding window
- **Time Limiter:** 5-second timeout on all routes
- **CORS:** Handled via Spring Cloud Gateway default filters
- **Deduplication:** `DedupeResponseHeader=Access-Control-Allow-Origin`
- **Health:** `/actuator/health` (aggregates all routes)

## 📡 Full API Endpoint Reference

### 1. Auth Service (`:8081`)

> **Live URL:** `https://pawfinder-auth-service.onrender.com`

| Method | Path | Description | Auth Required |
|--------|------|-------------|---------------|
| `POST` | `/api/v1/auth/register` | Register new user (email + password) | No |
| `POST` | `/api/v1/auth/verify` | Verify phone/SMS code | No |
| `POST` | `/api/v1/auth/email/register` | Register via email + password | No |
| `POST` | `/api/v1/auth/email/login` | Login via email + password | No |
| `POST` | `/api/v1/auth/login` | Login via OAuth provider (Google ID token) or email | No |
| `POST` | `/api/v1/auth/refresh` | Refresh access token using refresh token | Refresh Token |
| `GET` | `/api/v1/auth/me` | Get current authenticated user profile | Bearer JWT |

**Auth Providers Supported:**
- `GOOGLE` — Google OAuth 2.0 Sign-In
- `PHONE` — Twilio SMS verification
- `EMAIL` — Email + password (BCrypt hashed)

**JWT Configuration:**
- Access token expiration: 24 hours (`86400000` ms)
- Refresh token expiration: 7 days (`604800000` ms)
- Algorithm: HS256
- Secret: Configurable via `JWT_SECRET` env var

## 🔐 Authentication Flow

```
1. Client (Flutter Web) → Google Sign-In
2. Client receives Google ID token
3. Client → POST /api/v1/auth/login { authProvider: "GOOGLE", authId: "<google_id>" }
4. Auth Service validates, creates user if new, returns:
   {
     "accessToken": "eyJ... (24h)",
     "refreshToken": "eyJ... (7d)",
     "user": { "id": "uuid", "displayName": "...", ... }
   }
5. Client stores JWT in AuthCubit (in-memory + localStorage)
6. All subsequent requests include: Authorization: Bearer <accessToken>
7. Gateway validates JWT, extracts user context, forwards to service
```

**Token Refresh Flow:**
```
Client detects 401 → POST /api/v1/auth/refresh { refreshToken: "..." }
→ New access token returned → retry original request
```

| `Dockerfile.reward-service` | Reward Service |
| `Dockerfile.media-service` | Media Service |
| `Dockerfile.analytics-service` | Analytics Service |
| `Dockerfile.matching-service` | Matching Service |
| `Dockerfile.notification-service` | Notification Service |
| `Dockerfile` | Generic (local use, supports `--build-arg SERVICE_NAME`) |

All Dockerfiles use a **two-stage build**:
1. **Stage 1:** `maven:3.9-eclipse-temurin-21-alpine` — builds shared module + target service
2. **Stage 2:** `eclipse-temurin:21-jre-alpine` — minimal runtime, non-root user, health check

## 🏥 Health & Monitoring

All services expose Spring Boot Actuator:

- **Health:** `GET /actuator/health`
- **Info:** `GET /actuator/info`
- **Gateway routes:** `GET /actuator/gateway/routes` (gateway only)

### Known Health Check Issue

- **Redis health check disabled:** `MANAGEMENT_HEALTH_REDIS_ENABLED=false` is set to prevent intermittent failures caused by Render DNS not resolving the Upstash hostname (`tough-kangaroo-68983.upstash.io`) consistently.
- This does **not** affect rate limiting — the Lettuce Redis client handles reconnection internally.

| `MATCHING_SERVICE_URL` | Matching service URL |
| `NOTIFICATION_SERVICE_URL` | Notification service URL |

| Variable | Purpose |
